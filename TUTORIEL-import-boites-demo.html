Importer les boîtes de démo dans votre messagerie

Des mails 100 % factices et anonymisés, pour tester le tri et la rédaction sans jamais toucher à vos vrais mails. Une boîte par métier, à intégrer dans le client de votre choix — ou à lire directement, sans rien importer.

→Le principe

✅ But : travailler avec le connecteur mail
On importe ces boîtes de démo dans votre messagerie (dans un dossier de test dédié), puis Claude Cowork les lit via le connecteur mail — comme avec vos vrais e-mails, mais sur des données 100 % factices. Suivez la section correspondant à votre client mail ci-dessous.

1Les formats fournis & quel client les accepte

Trois formats sont livrés dans le dossier _distribution (et les .mbox dans _import_mac_mail) :

  • .eml — un fichier = un mail. Idéal pour le glisser-déposer dans les clients bureau.
  • .mbox — une boîte entière en un fichier. Format d'import d'Apple Mail et de Thunderbird.
  • .zip — les .eml regroupés par métier (à décompresser avant usage).

2Apple Mail macOS

  1. Décompressez TOUS_metiers_mbox.zip (ou repérez le dossier _import_mac_mail).
  2. Dans Mail : menu Boîte aux lettres → Importer des boîtes aux lettres…
  3. Choisissez « Fichiers au format mbox », puis Continuer.
  4. Sélectionnez le dossier contenant les .mbox (ou un fichier précis, ex. 03_Sales_Reception.mbox).
  5. Mail crée un dossier Import dans la barre latérale. Glissez-le sous « Sur mon Mac » et renommez-le (ex. « Démo Sales »).

Variante glisser-déposer : créez une boîte locale (Sur mon Mac → clic droit → Nouvelle boîte), puis faites glisser les .eml décompressés dedans.

3Outlook (application bureau) Windows Mac

Vrai pour l'Outlook classique (pas le « nouvel Outlook » — voir section 5).

  1. Dans Outlook, créez un dossier de test : clic droit sur votre boîte → Nouveau dossier (ex. « Démo Cowork »).
  2. Ouvrez l'Explorateur (Windows) ou le Finder (Mac) sur les .eml décompressés.
  3. Sélectionnez les fichiers et faites-les glisser directement dans le dossier « Démo Cowork » d'Outlook.
  4. Ils apparaissent comme des mails reçus normaux, prêts pour le test.
Astuce
Si Outlook est relié à un compte Exchange / Microsoft 365, les mails glissés dans un dossier en ligne se synchronisent aussi vers la version web du même compte.

4Thunderbird universel · gratuit

Thunderbird (gratuit, Windows & Mac) est le couteau suisse : il lit .eml et .mbox, et sert aussi de « pont » vers n'importe quel compte (section 6).

  1. Installez le module ImportExportTools NG (Menu → Modules → chercher « ImportExportTools »).
  2. Créez un dossier local : clic droit sur Dossiers locaux → Nouveau dossier.
  3. Clic droit sur ce dossier → ImportExportTools NG → Importer des messages (pour des .eml) ou Importer un fichier mbox.
  4. Sélectionnez les fichiers du métier voulu. Les mails apparaissent dans le dossier.

5Nouveau Outlook & Outlook Web navigateur / app récente

Ces versions n'ont pas d'import de fichiers. Pour intégrer les mails au compte, utilisez la méthode « pont » Thunderbird (section suivante) : elle pousse les mails vers le compte via IMAP — ils apparaissent alors dans le nouvel Outlook / Outlook Web, où le connecteur Cowork les lit.

6Gmail (web) & méthode « pont » universelle marche partout

Gmail web n'importe pas non plus de fichiers .eml/.mbox. La méthode « pont » fait apparaître les mails dans n'importe quel compte (Gmail, Outlook, autre) en passant par Thunderbird et la synchronisation IMAP :

  1. Dans Thunderbird, ajoutez le compte mail du participant (Gmail / Outlook…) en IMAP (Thunderbird configure tout automatiquement à partir de l'adresse).
  2. Importez les .eml/.mbox dans un dossier local (section 4).
  3. Dans le compte IMAP, créez un dossier « Démo Cowork ».
  4. Glissez (ou copiez) les mails du dossier local vers ce dossier « Démo Cowork » du compte IMAP.
  5. Thunderbird téléverse les mails sur le serveur : ils apparaissent dans Gmail / Outlook web, sur tous les appareils.
⚠️ Gmail & sécurité
Pour ajouter un compte Gmail à Thunderbird, la connexion se fait via la fenêtre Google habituelle (OAuth). Le participant saisit lui-même ses identifiants — ne les communiquez à personne.

7À retenir

  • Ces mails sont importés, pas « reçus » : ils n'arrivent pas par le réseau (c'est voulu — aucun vrai envoi).
  • Tout est factice et anonyme (clients A→F, références Ref-001→008, personnes fictives) : zéro donnée sensible.
  • Mettez-les dans un dossier de test dédié, jamais mélangés à de vrais mails.
  • Règle d'or maintenue : on ne laisse rien partir sans validation humaine, même en test.
  • Une fois importés, connectez votre messagerie à Cowork et ciblez le dossier de test : le connecteur mail fait le reste.
